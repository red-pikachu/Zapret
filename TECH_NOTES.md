# Zapret2Mac: Технические заметки и архитектура

Этот файл хранит историю технических решений, чтобы не сломать проект в будущем и понимать тонкости реализации под macOS.

## 1. Что это и как работает
**Zapret2Mac** — это утилита (Swift UI в статус-баре), оборачивающая движок `tpws` (от проекта zapret).
Она поддерживает два режима обхода DPI:
1. **Прозрачный прокси (для браузеров):**  
   Работает через скрипт `wrapper.sh`, который с помощью `sudo pfctl` заворачивает весь исходящий TCP-трафик на портах 80 и 443 на локальный порт `tpws` (988), который "ломает" пакеты (fragmentation, disorder, fake SNI) и отправляет их в сеть.
2. **SOCKS5-прокси (для Telegram и других клиентов):**  
   Поднимается отдельным процессом `tpws` с флагом `--socks` на порту 9988. Позволяет приложениям-клиентам обращаться к нему напрямую через настройки (socks5h://127.0.0.1:9988).

## 2. Важные ограничения macOS (Что НЕ НАДО делать)
macOS имеет крайне специфичный сетевой стек (Darwin/PF). Из-за этого многие фишки Linux здесь **не работают**:
- ❌ **НЕ ИСПОЛЬЗОВАТЬ `--oob` и `--disorder` одновременно.** В Linux это работает, но на macOS ядро не позволяет кидать out-of-band и out-of-order пакеты единовременно. Движок `tpws` просто **крашится** при запуске с ошибкой `in this system it's guaranteed to fail`. Оставляем только `--disorder`.
- ❌ **Network Extensions (VPN) конфликтуют с tpws.** Приложения вроде Happ Proxy (Hysteria 2) перехватывают трафик на уровне интерфейса (создают TUN-интерфейс `198.18.x.x`). Сырые и фрагментированные TCP-пакеты от `tpws` отторгаются таким адаптером с ошибкой `EPOLLHUP`. Запрет нужно использовать **при выключенном VPN**.
- ❌ **Не использовать скрипты Lua / `nfqws` (nfqueue).** На macOS нет подсистемы Netfilter (iptables). Вся логика может работать **только** через `pf` и `tpws`.

## 3. Особенности SOCKS5 и Telegram (MTProto)
По умолчанию движок `tpws` пытается быть умным и ищет в пакетах заголовки **HTTP** или **TLS (HTTPS)** (например, SNI). 
Telegram использует бинарный протокол **MTProto**, который выглядит как случайный мусор. `tpws` не распознает его и отправляет *без фрагментации*, из-за чего DPI его блокирует.
- ✅ **Обязательно** добавлять флаг `--split-any-protocol` для SOCKS-сервера Телеграма. Это заставляет `tpws` "ломать" трафик вслепую на 2-м байте (`--split-pos=2`), обходя блок!

## 4. Стратегии и их автообновление
Приложение умеет **бесшовно и автоматически обновлять стратегии обхода** DPI на запуске:
- Ссылки на актуальные стратегии берутся из ветки мастера GitHub проекта:  
  `https://raw.githubusercontent.com/red-pikachu/Zapret2Mac/main/strategies.json`
- Сама база стратегий собирается на основе актуальных обходов (например, адаптированные под `tpws` стратегии из Windows-репозиториев типа *Flowseal*).
- В `main.swift` метод `autoUpdateStrategies()` в фоне сверяет кэш. Если файл на GitHub изменился, он незаметно загружает его, обновляет меню на лету и выводит пуш-уведомление.

## 5. Запуск с правами Root без пароля
Чтобы прозрачный проксинг работал без постоянного запроса пароля от пользователя, в меню есть кнопка «Разрешить запуск без пароля».
Она единоразово добавляет `wrapper.sh` в `/private/etc/sudoers.d/zapret2_mac`. Если эту логику сломать, приложение будет постоянно запрашивать диалог авторизации Apple при клике на "Запустить".
